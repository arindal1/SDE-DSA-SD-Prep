# 🐸 Frog Jump [GFG [Link](https://www.geeksforgeeks.org/problems/geek-jump/1)]

This is one of the most important beginner-to-intermediate DP problems. It teaches how to think in terms of **minimum cost paths** instead of counting ways.



## ✅ Problem Statement

You are given an array `heights[]` of size `n`.

A frog starts at index `0` and wants to reach index `n - 1`.

At any position `i`, the frog can jump:

* to `i + 1`
* to `i + 2`

### Cost of a jump:

```text
abs(heights[i] - heights[j])
```

Return the **minimum total cost** to reach the last index.



## 🧠 Intuition

This is not about how many ways.

This is about **minimum cost**.

At any step `i`, the frog can come from:

* `i - 1`
* `i - 2`

So instead of thinking forward, think **backward**:

> “What is the cheapest way to reach step `i`?”



## 🔁 Recurrence Relation

```text
dp[i] = min(
    dp[i - 1] + abs(h[i] - h[i - 1]),
    dp[i - 2] + abs(h[i] - h[i - 2])
)
```



## 🧩 Brute Force Approach

Try all paths recursively.

```cpp
int solve(int i, vector<int>& h) {
    if (i == 0) return 0;

    int one = solve(i - 1, h) + abs(h[i] - h[i - 1]);

    int two = INT_MAX;
    if (i > 1)
        two = solve(i - 2, h) + abs(h[i] - h[i - 2]);

    return min(one, two);
}
```

### ❌ Problem

* Recomputes same states
* Time complexity: `O(2^n)`



## 🚀 Optimal Approach: Memoization



### 🧠 State Definition

```text
dp[i] = minimum cost to reach index i
```



### 🧮 Transition

```text
dp[i] = min(
    dp[i - 1] + cost(i, i - 1),
    dp[i - 2] + cost(i, i - 2)
)
```



### 🧱 Base Case

```text
dp[0] = 0
```



### 💾 Memoization Code

```cpp
class Solution {
public:
    int solve(int i, vector<int>& h, vector<int>& dp) {
        if (i == 0) return 0;

        if (dp[i] != -1) return dp[i];

        int one = solve(i - 1, h, dp) + abs(h[i] - h[i - 1]);

        int two = INT_MAX;
        if (i > 1)
            two = solve(i - 2, h, dp) + abs(h[i] - h[i - 2]);

        return dp[i] = min(one, two);
    }

    int frogJump(int n, vector<int>& heights) {
        vector<int> dp(n, -1);
        return solve(n - 1, heights, dp);
    }
};
```



## 🔍 Step-by-Step Example

### Input:

```text
heights = [10, 20, 30, 10]
```



### Build DP:

| i | Calculation       | dp[i] |
| - | -- | -- |
| 0 | base case         | 0     |
| 1 | abs(20-10)=10     | 10    |
| 2 | min(10+10, 0+20)  | 20    |
| 3 | min(20+20, 10+10) | 20    |



### ✅ Answer:

```text
20
```



## 📋 Tabulation (Bottom-Up)

```cpp
class Solution {
public:
    int frogJump(int n, vector<int>& h) {
        vector<int> dp(n, 0);

        dp[0] = 0;

        for (int i = 1; i < n; i++) {
            int one = dp[i - 1] + abs(h[i] - h[i - 1]);

            int two = INT_MAX;
            if (i > 1)
                two = dp[i - 2] + abs(h[i] - h[i - 2]);

            dp[i] = min(one, two);
        }

        return dp[n - 1];
    }
};
```



## 🚀 Space Optimization

Only last 2 states are needed.

```cpp
class Solution {
public:
    int frogJump(int n, vector<int>& h) {
        int prev2 = 0; // dp[i-2]
        int prev1 = 0; // dp[i-1]

        for (int i = 1; i < n; i++) {
            int one = prev1 + abs(h[i] - h[i - 1]);

            int two = INT_MAX;
            if (i > 1)
                two = prev2 + abs(h[i] - h[i - 2]);

            int curr = min(one, two);

            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
};
```



## 🧾 Complexity Analysis

| Approach        | Time   | Space |
| - | - | -- |
| Brute Force     | O(2^n) | O(n)  |
| Memoization     | O(n)   | O(n)  |
| Tabulation      | O(n)   | O(n)  |
| Space Optimized | O(n)   | O(1)  |



## 🧠 The Easiest Way to Remember

Think:

```text
To reach i, frog must come from i-1 or i-2
Pick the cheaper option
```

That’s it. No overthinking needed.

## 💡 Tips & Tricks

#### ✅ Tip 1: Always define DP meaning clearly

```text
dp[i] = min cost to reach i
```



### ✅ Tip 2: This is NOT counting problem

It is a **minimization problem**



### ✅ Tip 3: Always initialize carefully

Use `INT_MAX` for invalid paths



### ✅ Tip 4: Draw it like a graph

Each index is a node
Edges:

* i → i+1
* i → i+2

You are finding **shortest path in a linear DAG**



### ✅ Tip 5: Watch index bounds carefully

Most bugs happen here



## 🔄 Variations



### 🐸 [Frog Jump with K Steps](03.FrogJumpII.md)

Now frog can jump up to `k` steps.

#### Recurrence:

```text
dp[i] = min over j in [1...k] of:
dp[i - j] + abs(h[i] - h[i - j])
```



### 🐸 With Obstacles

Some positions blocked
→ skip those states



### 🐸 With energy limit

Track remaining energy
→ state becomes `dp[i][energy]`



## ❓ FAQs



### Q1. Why is dp[0] = 0?

Because we start there. No cost needed.



### Q2. Why use min and not sum?

Because we want **minimum cost**, not number of ways.



### Q3. Why do we check `i > 1`?

Because `i - 2` must be valid.



### Q4. Can this be solved greedily?

No. Local minimum doesn’t guarantee global minimum.



### Q5. Is this like Dijkstra?

Yes conceptually.
But since graph is linear, DP is enough.



## 📌 Final Summary

* This is a **1D DP minimization problem**
* Each state depends on last 2 states
* Transition:

  ```text
  dp[i] = min(dp[i-1], dp[i-2]) + cost
  ```
* Optimize space using 2 variables



## 🧠 Mental Model

* Climbing stairs → **count ways**
* Frog jump → **minimize cost**

Same structure
Different objective

That distinction is extremely important in DP.
