# 🐸 Frog Jump with K Steps [Article [Link](https://thejat.in/code/frog-jump-with-k-distances)]

This is the **natural extension** of the classic Frog Jump problem. Instead of only 1 or 2 jumps, the frog now has **K choices** at every step.

This problem is important because it introduces a **loop inside DP transitions**, which shows up frequently in harder problems.



## ✅ Problem Statement

You are given:

* An array `heights[]` of size `n`
* An integer `k`

A frog starts at index `0` and wants to reach index `n - 1`.

From any index `i`, the frog can jump to:

```text
i + 1, i + 2, ..., i + k
```

### Cost of a jump:

```text
abs(heights[i] - heights[j])
```

Return the **minimum total cost** to reach index `n - 1`.



## 🧠 Intuition

At any index `i`, the frog could have come from:

```text
i - 1, i - 2, i - 3, ..., i - k
```

So instead of two choices (like the basic frog jump), now we have **k choices**.



### 🔁 Recurrence Relation

```text
dp[i] = min over all j from 1 to k:
        dp[i - j] + abs(h[i] - h[i - j])
```

### Valid only if:

```text
i - j >= 0
```



## 🔍 Brute Force Approach

Try all possible jumps recursively.

```cpp
int solve(int i, vector<int>& h, int k) {
    if (i == 0) return 0;

    int ans = INT_MAX;

    for (int j = 1; j <= k; j++) {
        if (i - j >= 0) {
            int cost = solve(i - j, h, k) + abs(h[i] - h[i - j]);
            ans = min(ans, cost);
        }
    }

    return ans;
}
```



### ❌ Problem

* Recomputes same states multiple times
* Time complexity: **O(k^n)** in worst case

Not scalable.



## 🚀 Optimal Approach: Memoization



### 🧠 State Definition

```text
dp[i] = minimum cost to reach index i
```



### 🧮 Transition

```text
dp[i] = min(
    dp[i - 1] + cost,
    dp[i - 2] + cost,
    ...
    dp[i - k] + cost
)
```



### 🧱 Base Case

```text
dp[0] = 0
```



## 💾 Memoization Code

```cpp
class Solution {
public:
    int solve(int i, vector<int>& h, int k, vector<int>& dp) {
        if (i == 0) return 0;

        if (dp[i] != -1) return dp[i];

        int ans = INT_MAX;

        for (int j = 1; j <= k; j++) {
            if (i - j >= 0) {
                int cost = solve(i - j, h, k, dp) + abs(h[i] - h[i - j]);
                ans = min(ans, cost);
            }
        }

        return dp[i] = ans;
    }

    int frogJump(int n, vector<int>& heights, int k) {
        vector<int> dp(n, -1);
        return solve(n - 1, heights, k, dp);
    }
};
```



# 📋 Tabulation (Bottom-Up)

```cpp
class Solution {
public:
    int frogJump(int n, vector<int>& h, int k) {
        vector<int> dp(n, 0);

        dp[0] = 0;

        for (int i = 1; i < n; i++) {
            int ans = INT_MAX;

            for (int j = 1; j <= k; j++) {
                if (i - j >= 0) {
                    int cost = dp[i - j] + abs(h[i] - h[i - j]);
                    ans = min(ans, cost);
                }
            }

            dp[i] = ans;
        }

        return dp[n - 1];
    }
};
```

## 🧾 Complexity Analysis

### Time Complexity

```text
O(n * k)
```

* For each index `i`, we try up to `k` jumps



### Space Complexity

```text
O(n)
```

* DP array
* recursion stack (memoization)



## ⚠️ Important Insight About Space Optimization

Unlike the basic frog jump:

* Here, `dp[i]` depends on **k previous states**
* Not just 2

### ❌ So full space optimization is NOT possible

You cannot reduce to constant space unless `k` is small.



## 🧠 The Easiest Way to Remember

Think:

```text
From i, I can come from any of the last k steps
Pick the cheapest one
```

That’s literally the whole problem.



## 🪄 5-Step Intuition Trick

### 1. Define goal

Min cost to reach `i`



### 2. Identify options

From:

```text
i-1, i-2, ..., i-k
```



### 3. Write recurrence

```text
take minimum over all valid jumps
```



### 4. Add base case

```text
dp[0] = 0
```



### 5. Loop over k choices

This is the only new twist vs normal frog jump



## 💡 Tips & Tricks



### ✅ Tip 1: Always guard indices

```cpp
if (i - j >= 0)
```



### ✅ Tip 2: Initialize with INT_MAX

We are minimizing



### ✅ Tip 3: Think “window of k”

At every `i`, look back at last `k` states



### ✅ Tip 4: This is like sliding window DP

You are scanning previous `k` results



### ✅ Tip 5: Recognize pattern

Whenever you see:

* “you can jump up to k”
* “you can choose from last k states”

👉 Expect a loop in transition

## ❓ FAQs



### Q1. Why not greedy?

Because a locally cheaper jump may lead to a worse global cost.



### Q2. Why no constant space optimization?

Because we may need up to `k` previous values, not just 2.



### Q3. Can this be optimized further?

Yes, in special cases:

* using segment trees
* monotonic queues (advanced)



### Q4. Is this similar to knapsack?

Conceptually yes. You are choosing among multiple previous states.



### Q5. What if k = 2?

It becomes the **classic Frog Jump problem**



## 🧠 Mental Model

* Climbing Stairs → count ways
* Frog Jump → min over 2 choices
* Frog Jump K → min over **k choices**



## 📌 Final Summary

* This is a **1D DP minimization problem**
* Transition involves a **loop over k previous states**
* Time complexity becomes `O(n * k)`
* Cannot fully space optimize unless constraints are small



## 🧠 One-Line Takeaway

```text
dp[i] = best among last k positions + jump cost
```
