# 🔤 Word Ladder II [Leetcode #[126](https://leetcode.com/problems/word-ladder-ii/description/)]

Now we’ve entered the *director’s cut* version of [Word Ladder](09.WordLadder.md) 😄

In **[Word Ladder I](09.WordLadder.md)**, we only wanted the **length** of the shortest transformation.

In **Word Ladder II**, we want **all shortest paths**.

That changes the game.

Because now:

* BFS alone is not enough
* DFS alone is not enough
* we need a **combo move**:
  **BFS to find the shortest levels** + **DFS to rebuild all shortest sequences**

This is one of those problems that feels scary until you understand the structure.
Once it clicks, it’s actually beautiful. Like a Christopher Nolan plot, but with fewer suits and more strings 🎬



## 📌 Problem Statement

You are given:

* `beginWord`
* `endWord`
* `wordList`

You need to return **all shortest transformation sequences** from `beginWord` to `endWord` such that:

1. Only **one letter can be changed at a time**
2. Each transformed word must exist in `wordList`
3. Each sequence must be **as short as possible**

Return all such shortest sequences.



### ✅ Example

### Input

```text
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","dog","lot","log","cog"]
```

### Output

```text
[
  ["hit","hot","dot","dog","cog"],
  ["hit","hot","lot","log","cog"]
]
```

Because both are shortest valid transformation sequences.



## 🧠 First Big Realization

This is still a graph problem.

* each word = node
* one-letter transformation = edge

But now the question is not:

> “What is the shortest distance?”

Instead, it is:

> “What are **all** the shortest paths?”

That means:

* **BFS** is needed to find shortest distances
* but we must also remember **how we reached each word**
* and because a word can be reached from multiple parents at the same shortest level, we must store **all valid parents**

That’s the key twist 🔑



## 💡 Core Intuition

Think of BFS as building a shortest-path tree… except here it’s not a tree.

It’s more like a **shortest-path DAG**:

* each word may have multiple valid parents
* but only parents from the previous shortest level matter

Then after BFS is done:

* start from `endWord`
* go backward using parent links
* collect all possible shortest sequences

So the solution has 2 phases:

### Phase 1: BFS

Find shortest levels and store parent relationships

### Phase 2: DFS / Backtracking

Use the parent map to reconstruct every shortest path




## 🌱 Intuition in Plain English

Let’s say we found:

* `dog` can be reached from `dot`
* `log` can be reached from `lot`
* `cog` can be reached from both `dog` and `log`

Then `parents` would look like:

```text
parents["hot"] = ["hit"]
parents["dot"] = ["hot"]
parents["lot"] = ["hot"]
parents["dog"] = ["dot"]
parents["log"] = ["lot"]
parents["cog"] = ["dog", "log"]
```

Then DFS from `cog` gives:

* `cog -> dog -> dot -> hot -> hit`
* `cog -> log -> lot -> hot -> hit`

Reverse both:

* `hit -> hot -> dot -> dog -> cog`
* `hit -> hot -> lot -> log -> cog`

And that’s exactly what we want.



## 🐢 Brute Force Approach

A brute-force idea would be:

* generate all possible valid transformation sequences
* keep only the shortest ones

This is terrible because:

* number of paths can explode exponentially
* lots of repeated work
* impossible to scale well

So brute force is not practical.



## ⚡ Optimal Approach

The optimal approach is:

### BFS + Parent Tracking + DFS Reconstruction

### Why this works

* BFS guarantees shortest levels
* parent map preserves all shortest ways to reach each word
* DFS reconstructs all shortest paths using those parent links

That combination is the sweet spot.


## 🧾 Full Code (CPP)

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<vector<string>> ans;
    unordered_map<string, vector<string>> parents;
    unordered_map<string, int> level;

    void dfs(string word, string beginWord, vector<string>& path) {
        if (word == beginWord) {
            vector<string> temp = path;
            reverse(temp.begin(), temp.end());
            ans.push_back(temp);
            return;
        }

        for (auto &p : parents[word]) {
            path.push_back(p);
            dfs(p, beginWord, path);
            path.pop_back();
        }
    }

    vector<vector<string>> findLadders(string beginWord, string endWord, vector<string>& wordList) {
        ans.clear();
        parents.clear();
        level.clear();

        unordered_set<string> st(wordList.begin(), wordList.end());

        if (!st.count(endWord))
            return {};

        queue<string> q;
        q.push(beginWord);
        level[beginWord] = 0;

        while (!q.empty()) {
            string word = q.front();
            q.pop();

            int currLevel = level[word];
            string originalWord = word;

            for (int i = 0; i < word.size(); i++) {
                char original = word[i];

                for (char c = 'a'; c <= 'z'; c++) {
                    if (c == original) continue;

                    word[i] = c;

                    if (st.count(word)) {
                        if (!level.count(word)) {
                            level[word] = currLevel + 1;
                            q.push(word);
                            parents[word].push_back(originalWord);
                        } 
                        else if (level[word] == currLevel + 1) {
                            parents[word].push_back(originalWord);
                        }
                    }
                }

                word[i] = original;
            }
        }

        if (!level.count(endWord))
            return {};

        vector<string> path = {endWord};
        dfs(endWord, beginWord, path);

        return ans;
    }
};
```


## ✅ What our code does

Your code uses exactly that idea:

### During BFS:

* `level[word]` stores shortest distance from `beginWord`
* `parents[word]` stores all words from which `word` was reached at the shortest level

### During DFS:

* Start from `endWord`
* Recursively move through its parents
* Once `beginWord` is reached, reverse the path and store it

This is the standard and correct approach ✅



## 🧩 Step-by-Step Algorithm

### BFS phase

1. Put all words in a set
2. Start BFS from `beginWord`
3. Store:

   * `level[word] = shortest distance from beginWord`
   * `parents[word] = all words that can reach word at shortest level`
4. For every generated neighbor:

   * if first time seen, set level and parent
   * if seen again at same shortest level, add another parent

### DFS phase

5. Start from `endWord`
6. Move backward using `parents`
7. When `beginWord` is reached, reverse path and add to answer

## 📊 Complexity Analysis

Let:

* `N = number of words`
* `L = length of each word`

### BFS Time Complexity

For each processed word:

* try `L` positions
* try `26` replacements

So BFS is about:

`O(N * L * 26)`
which is usually written as:

`O(N * L)`



### DFS Complexity

This depends on the number of shortest paths.

If there are many shortest sequences, DFS must output all of them.

So total complexity is:

`O(N * L + total size of all output sequences)`

That output-dependent part is important.



### Space Complexity

* `level` map: `O(N)`
* `parents` map: can be up to `O(N^2)` in dense shortest-path structure
* recursion stack for DFS

So a fair answer is:

`O(N^2)` in worst case due to parent storage

or more carefully:

`O(N + total parent links)`

## 🎯 Interview Explanation

A clean interview explanation would be:

> Since we need all shortest transformation sequences, I first run BFS from `beginWord` to compute the shortest level at which each word can be reached. While doing BFS, I store for every word all its possible parents that reach it at the shortest distance. After BFS, if `endWord` is reachable, I use DFS from `endWord` back to `beginWord` using the parent map to reconstruct all shortest paths.

That is a very strong answer.

## 💡 Tips and Tricks

### 1. Do not store only one parent

That would lose valid shortest sequences.

For example:

```text
cog <- dog
cog <- log
```

Both matter.



### 2. Parent is added only if it gives shortest level

This condition is crucial:

```cpp
else if (level[word] == currLevel + 1)
```

That means:

* only shortest path parents are stored
* longer paths are ignored

Very important.



### 3. Clear class members

Since `ans`, `parents`, and `level` are class variables, always clear them inside `findLadders`.

This is one of those tiny things that saves you from weird bugs later.



### 4. BFS builds structure, DFS extracts answers

Nice way to mentally separate the problem.



### 5. Output size can be huge

Even with efficient BFS, the number of shortest sequences may be very large.
That’s unavoidable because the problem asks you to return all of them.

## ❓ FAQs

### Q1. Why not use only DFS?

Because DFS does not guarantee shortest paths and would explore way too many possibilities.



### Q2. Why not stop BFS as soon as `endWord` is first found?

You can stop after finishing that level, but not immediately at the first encounter, because there may be other shortest parents for `endWord` in the same BFS level.

That’s an important nuance.



### Q3. Why store levels?

To ensure we only keep shortest-path parent relationships.



### Q4. Why DFS from endWord instead of beginWord?

Because `parents` naturally points backward. Reconstructing backward is simpler.



### Q5. Can a word have multiple parents?

Yes, and that is exactly why this problem is harder than Word Ladder I.
