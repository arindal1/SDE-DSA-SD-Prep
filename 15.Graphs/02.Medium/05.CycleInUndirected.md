# ✅ Cycle Detection in an Undirected Graph [GFG [Link](https://www.geeksforgeeks.org/problems/detect-cycle-in-an-undirected-graph/1)] 




## Problem statement

Given `V` vertices (numbered `0` to `V-1`) and a list of undirected edges `edges` (each edge is a pair `{u, v}`), determine whether the graph contains **any cycle**.

**Important input note:** This code assumes **0-based** vertex indices. If your input is 1-based (common in some problems), subtract 1 from each vertex before using the algorithm.



## Intuition (in plain English)

Think of walking along the graph. While doing a DFS from any node, if you find a neighbor you've already visited and that neighbor is **not** the node you came from (the *parent*), then you've found a way back to an already-visited node by a different route → that's a cycle.

* For every connected component, start DFS if the node is unvisited.
* Track the parent so the immediate back-edge to parent doesn't count as a cycle.

This is like navigating a maze: if you find a previously visited room that did not directly lead you there, the maze has a loop. 🌀



## Brute force idea (not great, but intuitive)

* For each vertex, try to find a cycle by exploring all paths and checking if you return to a vertex via a different route.
* This leads to exploring exponentially many paths — too slow and unnecessary.

We prefer DFS with parent-tracking which is linear in edges+vertices.



## Optimal approach (what the code does)

Use DFS (recursive or iterative) with a `visited` array and a `parent` parameter:

1. Build adjacency list.
2. Maintain `visited[v]` (0/1 or bool).
3. For each node `i` not visited, call `detectDFS(i, parent = -1)`.
4. In `detectDFS(node, parent)`:

   * Mark `node` visited.
   * For each `neighbor` of `node`:

     * If `neighbor` not visited: recursively DFS; if it returns `true`, propagate `true`.
     * Else if `neighbor != parent`, return `true` (cycle detected).
5. If DFS never returns `true`, graph is acyclic.

This runs in `O(V + E)` time and `O(V + E)` space (adjacency + recursion stack).



# Full Code (clean, self-contained C++ with examples)

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    // Recursive DFS that returns true if cycle found starting from "node".
    bool detectDFS(int node, int parent, vector<int> &vis, const vector<vector<int>> &adj) {
        vis[node] = 1; // mark visited

        for (int neighbor : adj[node]) {
            if (!vis[neighbor]) {
                if (detectDFS(neighbor, node, vis, adj))
                    return true;
            }
            // If neighbor is visited and not the parent, we found a cycle
            else if (neighbor != parent) {
                return true;
            }
        }
        return false;
    }

public:
    // edges: list of pairs {u,v} with 0-based indices
    bool isCycle(int V, const vector<vector<int>> &edges) {
        // Build adjacency list
        vector<vector<int>> adj(V);
        for (auto &e : edges) {
            if (e.size() < 2) continue; // safety: skip invalid entries
            int u = e[0];
            int v = e[1];
            // assuming 0-based; if input is 1-based, convert: u--, v--;
            adj[u].push_back(v);
            adj[v].push_back(u);
        }

        vector<int> vis(V, 0);

        // For every component
        for (int i = 0; i < V; ++i) {
            if (!vis[i]) {
                if (detectDFS(i, -1, vis, adj))
                    return true;
            }
        }
        return false;
    }
};

// Example usage
int main() {
    Solution sol;

    // Test 1: simple triangle (0-1-2-0) -> cycle
    int V1 = 3;
    vector<vector<int>> edges1 = {{0,1},{1,2},{2,0}};
    cout << "Test1 (should be 1): " << sol.isCycle(V1, edges1) << "\n";

    // Test 2: tree (no cycle)
    int V2 = 4;
    vector<vector<int>> edges2 = {{0,1},{1,2},{1,3}};
    cout << "Test2 (should be 0): " << sol.isCycle(V2, edges2) << "\n";

    // Test 3: self-loop
    int V3 = 2;
    vector<vector<int>> edges3 = {{0,0}}; // self-loop -> cycle
    cout << "Test3 (should be 1): " << sol.isCycle(V3, edges3) << "\n";

    // Test 4: disconnected graph; one component has cycle
    int V4 = 5;
    vector<vector<int>> edges4 = {{0,1},{1,2},{2,0},{3,4}};
    cout << "Test4 (should be 1): " << sol.isCycle(V4, edges4) << "\n";

    return 0;
}
```



## Complexity analysis

* **Time complexity:** `O(V + E)`

  * Building adjacency: `O(E)`
  * DFS visits each vertex and edge once across all components → `O(V + E)`
* **Space complexity:** `O(V + E)` for adjacency list + `O(V)` for visited array + recursion stack up to `O(V)` in worst case.



## Edge cases & input assumptions (be explicit)

* **Indexing:** Code assumes **0-based**. If input edges are 1-based, convert with `u--, v--`.
* **Parallel edges:** Multiple edges between same vertices do not break the algorithm.
* **Self-loops:** Detects them as cycles.
* **Disconnected graph:** The `for` loop ensures every component gets checked.
* **Invalid edge entries:** I added a safety `if (e.size() < 2) continue;` — better to validate input before calling.



## Alternate approaches (nice to know in interviews)

### 1) Union-Find (Disjoint Set Union — DSU) — very popular

* For each edge `(u,v)`:

  * If `find(u) == find(v)` → adding this edge connects two vertices already in the same component ⇒ cycle.
  * Else `union(u, v)`.
* **Time:** near `O(E α(V))` (α is inverse Ackermann — almost constant)
* Good when edges are processed one by one or for Kruskal-like logic.

**Short DSU sketch**:

```cpp
struct DSU {
    vector<int> parent, rank;
    DSU(int n) : parent(n), rank(n,0) { for(int i=0;i<n;i++) parent[i]=i; }
    int find(int x){ return parent[x]==x?x:parent[x]=find(parent[x]); }
    bool unite(int a,int b){
        a=find(a); b=find(b); if(a==b) return false; // already same set
        if(rank[a]<rank[b]) swap(a,b);
        parent[b]=a;
        if(rank[a]==rank[b]) rank[a]++;
        return true;
    }
};
```

### 2) Iterative DFS/BFS with parent tracking

* Use a stack or queue; store `(node, parent)`.
* Useful to avoid recursion depth on huge graphs.

## The easiest way to remember this algorithm in interviews

**"DFS + parent = cycle check"**
When using DFS in an undirected graph, always pass the parent of the current node: if you see a visited neighbor that isn’t the parent, it's a cycle.

Short mnemonic: **DFS → visited? → parent? → cycle!** 🧭



## A 5-step intuition trick so you never forget this problem

1. **Think walk** — you are walking from node to node (DFS).
2. **Mark home base** — mark nodes visited when you enter them.
3. **Remember who gave you the map** — pass the parent node.
4. **If you see a place you’ve seen before and it’s not who showed you here → loop**.
5. **Do this for each island (component)** — start DFS if unvisited.

If you translate to code: `visited[node] = true; for neighbor: if !visited => dfs; else if neighbor != parent => cycle`.



## Tips, tricks & variations

* **If recursion depth is an issue**, use iterative DFS or BFS with parent-pair tracking.
* **Union-Find** is often the shortest approach in code and excellent for streaming edges.
* **Directed graphs** require different logic (use colors / recursion stack detection).
* **Multiple edges** between same nodes don't falsely create cycles in this method.
* **Graph input format**: always clarify whether nodes are 0-based or 1-based — a frequent trap in interviews/competitions.
* **Print helpful debug info** while developing (component id, parent mapping) — then remove for final submission.



## FAQs (common interviewer / user questions)

**Q: Does this work for directed graphs?**

A: No — for directed graphs you must detect back edges using a recursion stack or color method (white/gray/black), not parent-based undirected logic.

**Q: What about parallel edges?**

A: The algorithm handles them. Parallel edges do not create a false positive because the parent check filters out the trivial back-and-forth. Self-loop is correctly detected as a cycle.

**Q: What if nodes are 1..V?**

A: Convert to 0-based by subtracting 1 from vertices when building adj list.

**Q: Is recursion safe for large V?**

A: If `V` could be up to 10⁵, recursion may overflow stack. Use iterative DFS or increase stack size, or use DSU depending on constraints.

**Q: Which is better — DFS or Union-Find?**

A: Both are valid. Union-Find is simpler if you only need to check edges one-by-one. DFS gives more structure if you might also want to find connected components or do other graph traversals.



## Variations & interview extensions

* **Return the actual cycle path**: When you detect cycle in DFS, track parent chain and reconstruct nodes from current to neighbor.
* **Count number of cycles**: More complex; must handle counting unique cycles (not trivial).
* **Detect cycles and components** simultaneously — combine with component labeling.



## Quick cheatsheet (copy-paste for interviews)

* Build adjacency list → `O(V + E)`.
* `vector<int> vis(V, 0)`; call `dfs(node, parent=-1)` for each unvisited node.
* In DFS: for `neighbor`:

  * if not visited → `dfs(neighbor, node)`
  * else if neighbor != parent → cycle → return true
* Complexity: `O(V + E)` time, `O(V + E)` space.
