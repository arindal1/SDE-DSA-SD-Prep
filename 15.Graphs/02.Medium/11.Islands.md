## 📌 Problem Statement

You are given a 2D grid of size `n x m` where:

* `'1'` = land
* `'0'` = water

An island is formed by connecting adjacent land cells (4 directions only).

👉 Return the **number of islands**.



## 🧠 Intuition (Simple & Powerful)

Think of the grid like a map 🗺️

Every time you see a `'1'`, it could be:

* part of an already discovered island
* or the start of a **new island**

So:

> “Whenever you find an unvisited `'1'`, you’ve found a new island”

Then you:

* explore the entire island using DFS/BFS
* mark all its cells as visited
* move on



## 💡 Core Idea

### Count how many times you start a DFS

Each DFS = one island

That’s it.

## 🐢 Brute Force Approach

A naive idea:

* For each cell:

  * try exploring without marking visited globally
  * count components

But this leads to:

* repeated traversals
* unnecessary work

Worst-case complexity can blow up.



## ⚡ Optimal Approach

### DFS / BFS Traversal

### Steps:

1. Traverse grid
2. If `'1'` and not visited:

   * increment count
   * run DFS/BFS
3. DFS marks entire island visited


## 🧾 Our Code (CPP + DFS)


```cpp id="r0z7iv"
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    void dfs(int row, int col, vector<vector<char>>& grid, vector<vector<int>>& vis) {
        vis[row][col] = 1;

        int n = grid.size(), m = grid[0].size();

        int drow[] = {-1, 0, 1, 0};
        int dcol[] = {0, 1, 0, -1};

        for (int i = 0; i < 4; ++i) {
            int nrow = row + drow[i];
            int ncol = col + dcol[i];

            if (nrow >= 0 && nrow < n &&
                ncol >= 0 && ncol < m &&
                !vis[nrow][ncol] &&
                grid[nrow][ncol] == '1') {

                dfs(nrow, ncol, grid, vis);
            }
        }
    }

public:
    int numIslands(vector<vector<char>>& grid) {
        int n = grid.size();
        if (n == 0) return 0;

        int m = grid[0].size();

        vector<vector<int>> vis(n, vector<int>(m, 0));
        int count = 0;

        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < m; ++j) {
                if (!vis[i][j] && grid[i][j] == '1') {
                    count++;
                    dfs(i, j, grid, vis);
                }
            }
        }

        return count;
    }
};
```

## 🔧 Optimization 1: Remove the `vis` array (in-place modification)

Instead of using a separate `vis` matrix, you can **mark visited cells directly in the grid** by changing `'1' → '0'`.

### ✅ Why this helps:

* Saves `O(n × m)` extra space
* Cleaner code
* Same time complexity

### 💡 Optimized Code:

```cpp
class Solution {
private:
    void dfs(int row, int col, vector<vector<char>>& grid) {
        int n = grid.size(), m = grid[0].size();
        grid[row][col] = '0'; // mark visited

        int drow[] = {-1, 0, +1, 0};
        int dcol[] = {0, +1, 0, -1};

        for (int i = 0; i < 4; ++i) {
            int nrow = row + drow[i];
            int ncol = col + dcol[i];

            if (nrow >= 0 && nrow < n && ncol >= 0 && ncol < m &&
                grid[nrow][ncol] == '1') {
                dfs(nrow, ncol, grid);
            }
        }
    }

public:
    int numIslands(vector<vector<char>>& grid) {
        int n = grid.size();
        if (n == 0) return 0;
        int m = grid[0].size();

        int count = 0;

        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < m; ++j) {
                if (grid[i][j] == '1') {
                    count++;
                    dfs(i, j, grid);
                }
            }
        }

        return count;
    }
};
```



## ⚡ Optimization 2: Use BFS instead of DFS (avoid recursion stack overflow)

If the grid is very large, recursive DFS may cause **stack overflow**.

### 🔁 BFS Alternative:

```cpp
void bfs(int row, int col, vector<vector<char>>& grid) {
    int n = grid.size(), m = grid[0].size();
    queue<pair<int,int>> q;
    q.push({row, col});
    grid[row][col] = '0';

    int drow[] = {-1, 0, +1, 0};
    int dcol[] = {0, +1, 0, -1};

    while (!q.empty()) {
        auto [r, c] = q.front(); q.pop();

        for (int i = 0; i < 4; ++i) {
            int nr = r + drow[i];
            int nc = c + dcol[i];

            if (nr >= 0 && nr < n && nc >= 0 && nc < m &&
                grid[nr][nc] == '1') {
                grid[nr][nc] = '0';
                q.push({nr, nc});
            }
        }
    }
}
```



## 🧠 Optimization 3: Union-Find (Advanced)

We *can* use **Disjoint Set (Union-Find)**:

* Same time complexity
* More complex code
* Useful if you need dynamic connectivity

👉 Not necessary for this problem unless explicitly asked.

| Approach     | Time  | Space | Notes               |
| -- | -- | -- | - |
| Our DFS     | O(nm) | O(nm) | Good                |
| In-place DFS | O(nm) | O(1)  | ✅ Best practical    |
| BFS          | O(nm) | O(nm) | Safe for deep grids |
| Union-Find   | O(nm) | O(nm) | Overkill            |

## 🪄 5-Step Intuition Trick

### 1. Grid = graph

Cells are nodes.

### 2. Land = valid nodes

Only `'1'` matters.

### 3. Connected land = island

DFS finds connected component.

### 4. Start DFS only once per island

That’s your count.

### 5. Mark visited

So you don’t double count.

Done.

## ❓ FAQs

### Q1. Why DFS works?

Because an island is just a connected component.



### Q2. Why mark visited?

To avoid counting the same island multiple times.



### Q3. Can BFS be used?

Yes, exact same logic.



### Q4. What about diagonal connections?

Not allowed here (only 4 directions).



#3# Q5. What if grid is huge?

Prefer BFS or iterative DFS to avoid stack overflow.
