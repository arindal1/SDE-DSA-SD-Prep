# 🔁 Cycle Detection in Directed Graph (DFS + Recursion Stack) [GFG [Link](https://www.geeksforgeeks.org/problems/detect-cycle-in-a-directed-graph/1)]

This one is a **core graph pattern** and shows up everywhere:

* Course Schedule
* Dependency resolution
* Build systems
* Topological sorting

At first glance, it feels similar to undirected cycle detection…
but the trick here is **very different**.



## 📌 Problem Statement

Given:

* `V` vertices (`0 → V-1`)
* a list of directed edges `{u → v}`

👉 Determine whether the graph contains a **cycle**.



## 🧠 Intuition (The Real Difference)

In **undirected graphs**, we used:

> “visited + parent”

But here, that **does NOT work** ❌

Because direction matters.



### 💡 Key Idea

> A cycle exists if we revisit a node that is **currently in our DFS path**

This is called a **back edge**.



## 🔥 Core Concept

We use **two arrays**:

### 1. `vis[]`

* Marks nodes we have ever visited

### 2. `path[]` (recursion stack)

* Marks nodes currently in DFS path



### 🧠 Why do we need `path[]`?

Because:

* Visiting a node again is not always a cycle
* It is only a cycle if that node is **still in the current recursion path**

## 🐢 Brute Force Idea

Try all possible paths and check if any loops back.

### Why bad?

* exponential paths
* repeated work
* inefficient



## ⚡ Optimal Approach

### DFS + Recursion Stack

### Steps:

1. Build adjacency list
2. For each node:

   * if not visited → DFS
3. In DFS:

   * mark visited
   * mark in path
   * explore neighbors
   * if neighbor is in path → cycle
4. remove node from path when done




## ✅ What our Code Does

Our code uses:

* DFS traversal
* `vis[]` → visited nodes
* `path[]` → recursion stack

### Logic:

1. Mark node visited
2. Mark node in current path
3. Explore neighbors:

   * If not visited → DFS
   * If visited AND in path → cycle
4. Remove node from path before returning

This is the **correct and optimal approach** ✅



## 🧾 Our Code (Clean + Explained)

```cpp id="x3p9h7"
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    bool dfs(int node, vector<vector<int>>& adj, vector<int>& vis, vector<int>& path) {
        vis[node] = 1;
        path[node] = 1;

        for (auto it : adj[node]) {
            if (!vis[it]) {
                if (dfs(it, adj, vis, path))
                    return true;
            } 
            else if (path[it]) {
                // back edge detected
                return true;
            }
        }

        // remove from recursion stack
        path[node] = 0;
        return false;
    }

public:
    bool isCyclic(int V, vector<vector<int>> &edges) {
        vector<vector<int>> adj(V);

        // build adjacency list
        for (auto &e : edges) {
            int u = e[0];
            int v = e[1];
            adj[u].push_back(v);
        }

        vector<int> vis(V, 0);
        vector<int> path(V, 0);

        // handle disconnected components
        for (int i = 0; i < V; i++) {
            if (!vis[i]) {
                if (dfs(i, adj, vis, path))
                    return true;
            }
        }

        return false;
    }
};
```

## ❓ FAQs

### Q1. Why can’t we use parent like undirected graph?

Because direction matters. Parent doesn’t define cycle here.



### Q2. Why do we need `path[]`?

To detect if a node is part of current DFS recursion.



### Q3. What is a back edge?

An edge pointing to an ancestor in DFS tree.



### Q4. Can BFS be used?

Yes → Kahn’s Algorithm (indegree method)



### Q5. What if graph is disconnected?

Handled by looping over all nodes.



## ✨ BFS Alternative (Kahn’s Algorithm)

```cpp id="kq1d8v"
bool isCyclic(int V, vector<vector<int>>& edges) {
    vector<vector<int>> adj(V);
    vector<int> indegree(V, 0);

    for (auto &e : edges) {
        adj[e[0]].push_back(e[1]);
        indegree[e[1]]++;
    }

    queue<int> q;

    for (int i = 0; i < V; i++) {
        if (indegree[i] == 0)
            q.push(i);
    }

    int count = 0;

    while (!q.empty()) {
        int node = q.front(); q.pop();
        count++;

        for (auto it : adj[node]) {
            indegree[it]--;
            if (indegree[it] == 0)
                q.push(it);
        }
    }

    return count != V;
}
```



## 🧠 One-line Memory Hook

> **If DFS revisits a node still in the recursion path → cycle exists**
