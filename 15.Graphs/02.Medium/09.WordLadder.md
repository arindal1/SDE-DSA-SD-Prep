# 🔤 Word Ladder [Leetcode #[127](https://leetcode.com/problems/word-ladder/description/)]

Ah, this one is a classic.
A very *“looks like strings, is actually a graph”* problem 😄

And once you see that, everything becomes much cleaner.

This is not really about words.
It’s about **shortest path in an unweighted graph**.

That’s the whole secret sauce 🍝



# 📌 Problem Statement

You are given:

* a `beginWord`
* an `endWord`
* a `wordList`

You need to transform `beginWord` into `endWord` such that:

1. You can change **only one letter at a time**
2. Each transformed word must exist in `wordList`
3. Return the **length of the shortest transformation sequence**

If no such transformation exists, return `0`.



### ✅ Example

### Input

```text
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","dog","lot","log","cog"]
```

### Output

```text
5
```

### Explanation

One shortest path is:

```text
hit → hot → dot → dog → cog
```

This sequence has 5 words, so answer is `5`.



## 🧠 First Big Realization

This is a graph problem in disguise.

Each word is like a node.

There is an edge between two words if they differ by **exactly one character**.

So the problem becomes:

> Find the shortest path from `beginWord` to `endWord`

And because every valid transformation has equal cost, the best tool is:

## 🚀 BFS

Because BFS gives shortest path in an **unweighted graph**.

That’s the big interview insight.



## 💡 Core Intuition

Think of each word as being on the same social network.

A word can connect only to words that differ by one character.

Example:

```text
hit
│
hot
├── dot
└── lot
```

You want the shortest route from `beginWord` to `endWord`.

That is exactly what BFS does best:

* explore level by level
* first time reaching target = shortest path

It’s basically the LinkedIn of words, but with more panic and fewer endorsements 😌



## 🔑 Why BFS and not DFS?

Because DFS can find *a* path, but not necessarily the shortest one.

BFS explores all words reachable in:

* 1 step
* then 2 steps
* then 3 steps

So the first time we reach `endWord`, we are guaranteed to have found the shortest sequence.



## ✅ What our code does

Our code performs BFS where:

* queue stores `{currentWord, steps}`
* each step tries changing every character to every letter `'a'` to `'z'`
* if the new word exists in the set, it is added to the queue
* once a word is used, it is removed from the set to avoid revisiting

This is the **standard optimal BFS solution** ✅



## 🧾 Our Code 


```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int ladderLength(string beginWord, string endWord, vector<string>& wordList) {
        queue<pair<string, int>> q;
        q.push({beginWord, 1});

        unordered_set<string> st(wordList.begin(), wordList.end());

        // Optional optimization:
        // If endWord is not even present, no valid transformation exists
        if (st.find(endWord) == st.end()) return 0;

        // Remove beginWord if present, to avoid revisiting it
        st.erase(beginWord);

        while (!q.empty()) {
            string word = q.front().first;
            int steps = q.front().second;
            q.pop();

            if (word == endWord)
                return steps;

            for (int i = 0; i < word.size(); ++i) {
                char original = word[i];

                for (char c = 'a'; c <= 'z'; ++c) {
                    word[i] = c;

                    if (st.find(word) != st.end()) {
                        q.push({word, steps + 1});
                        st.erase(word); // mark visited
                    }
                }

                word[i] = original; // restore original character
            }
        }

        return 0;
    }
};
```




## ⚙️ Step-by-Step Algorithm

### 1. Put all words into a hash set

This gives `O(1)` average lookup.

### 2. Start BFS from `beginWord`

Queue stores:

* current word
* current number of steps in transformation

### 3. For each word popped from queue:

Try changing each position one by one.

If word is `"hit"`:

* change index 0: `ait`, `bit`, `cit`, ...
* change index 1: `hat`, `hbt`, `hct`, ...
* change index 2: `hia`, `hib`, `hic`, ...

### 4. If generated word exists in set:

* push it into queue
* remove it from set so it isn't revisited

### 5. If you reach `endWord`

Return number of steps.

### 6. If BFS finishes without reaching target

Return `0`

## 🐢 Brute Force Approach

A naive idea would be:

* Build all possible transformation sequences
* Try all paths recursively
* Track minimum length

That becomes extremely expensive because:

* branching factor can be huge
* many repeated states
* exponential explosion

### Time Complexity

Brute force can go exponential in the number of words.

Absolutely not the move.



## ⚡ Optimal Approach

The optimal approach is **BFS + hash set**.

### Why optimal?

Because:

* BFS guarantees shortest path in unweighted graph
* hash set gives fast lookup for valid transformed words
* each word is processed at most once



## 📊 Complexity Analysis

Let:

* `N = number of words in wordList`
* `L = length of each word`

### Time Complexity

### `O(N * L * 26)`

More precisely:

* each word can be processed once
* for each word:

  * for each character position `L`
  * try all 26 letters

So total is approximately:

`O(N * L * 26)`
which is usually written as:

`O(N * L)`

since 26 is constant.



### Space Complexity

### `O(N)`

Because of:

* hash set
* BFS queue

If you count string storage fully, you can say `O(N * L)` depending on detail level, but in most interviews `O(N)` is fine.

## 💡 Tips and Tricks

### 1. The graph is implicit

You do **not** build the full graph beforehand.

That would be more expensive.

Instead, generate neighbors when needed.

That’s a big optimization.



### 2. Use `unordered_set`

This gives fast membership checking and deletion.

Very important for performance.



### 3. Remove from set as soon as you push

This is important.

Why not remove after pop?

Because if multiple paths discover the same word before it is popped, it could get added to queue multiple times.

Removing immediately avoids duplicates.



### 4. Don’t forget to restore the original character

After trying all replacements for a position, put the character back.

Otherwise your next transformations will be based on the wrong word.

This line matters:

```cpp
word[i] = original;
```

Tiny detail, huge correctness.



### 5. This pattern appears a lot

Whenever you see:

* states
* transformations
* shortest sequence
* equal move cost

Think BFS.

## ❓ FAQs

### Q1. Why do we use BFS?

Because every valid transformation has equal cost, and BFS guarantees shortest path in unweighted graphs.



### Q2. Why remove words from the set?

To mark them as visited and prevent repeated processing.



### Q3. Why remove immediately when pushing, not later?

Because otherwise the same word might be enqueued multiple times from different parents in the same BFS level.



### Q4. Can DFS work?

DFS can find a path, but it does not guarantee the shortest one.



### Q5. Why not compare current word with every word in wordList to find neighbors?

That would be too slow.

Instead, generate possible neighbors by changing one character at a time.

That’s much more efficient.



## ✨ Slightly More Polished Version

Here’s the same solution with the helpful early check:

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int ladderLength(string beginWord, string endWord, vector<string>& wordList) {
        unordered_set<string> st(wordList.begin(), wordList.end());

        if (st.find(endWord) == st.end())
            return 0;

        queue<pair<string, int>> q;
        q.push({beginWord, 1});

        st.erase(beginWord);

        while (!q.empty()) {
            auto [word, steps] = q.front();
            q.pop();

            if (word == endWord)
                return steps;

            for (int i = 0; i < word.size(); ++i) {
                char original = word[i];

                for (char c = 'a'; c <= 'z'; ++c) {
                    word[i] = c;

                    if (st.find(word) != st.end()) {
                        q.push({word, steps + 1});
                        st.erase(word);
                    }
                }

                word[i] = original;
            }
        }

        return 0;
    }
};
```
