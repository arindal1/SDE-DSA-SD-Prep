# 🧠 Surrounded Regions [Leetcode #[130](https://leetcode.com/problems/surrounded-regions/description/)]

This is one of those grid problems where the trick is **not** to focus on what to flip first… but on what **not** to flip.
That’s the whole game here 😌

At first glance, it feels like:

> “Find every region of `O` fully surrounded by `X`”

But doing that directly gets messy.

The smarter idea is:

> “Any `O` connected to the boundary can never be surrounded.”

So instead of hunting trapped cells, we **protect the safe ones first**.
Everything else gets flipped.

That mental shift is the key 🔑



## 📌 Problem Statement

You are given a 2D board containing:

* `'X'`
* `'O'`

Your task is to **capture all regions surrounded by `X`**.

A region is captured by flipping all `'O'`s into `'X'`s in that surrounded region.

### Important rule

An `'O'` should **not** be flipped if:

* it is on the boundary, or
* it is connected to a boundary `'O'` through adjacent `'O'` cells

Adjacency is only in **4 directions**:

* up
* right
* down
* left

### Input:
    [["X","X","X","X"],
     ["X","O","O","X"],
     ["X","X","O","X"],
     ["X","O","X","X"]]

### Output: 

    [["X","X","X","X"],
     ["X","X","X","X"],
     ["X","X","X","X"],
     ["X","O","X","X"]]

### Explanation:

![img](https://assets.leetcode.com/uploads/2021/02/19/xogrid.jpg)

In the above diagram, the bottom region is not captured because it is on the edge of the board and cannot be surrounded.

## ✅ What we are doing...

Our code is using **DFS from boundary `O`s**.

That means:

1. Visit all `O`s on the border
2. Mark every `O` connected to those boundary `O`s as **safe**
3. After that, any remaining `O` must be surrounded
4. Flip those remaining `O`s to `X`

This is the standard and correct optimal solution 👏



## 🌱 Core Intuition

Imagine the board like a city under siege 🏰

* Boundary `O`s have an escape route
* Any `O` connected to them also has an escape route
* Only the `O`s trapped completely inside are doomed 😅

So the strategy becomes:

### Don’t search for trapped cells directly

Search for **safe cells first**

That’s way easier.



## ❌ Wrong First Thought

A lot of people initially think:

> “Let me DFS every `O` and check if that component touches the border”

That works, but it’s clunky:

* repeated exploration
* more bookkeeping
* more chances to mess up in an interview

>Instead:

### ✅ Better approach

Start from border `O`s and mark all reachable `O`s as safe.

Then flip the rest.

Very clean. Very elegant. Very “I know what I’m doing” energy 😌



## 🔍 Example


### Step 1: Look at boundary `O`s

Only this one is on the boundary:

```text
X X X X

X O X X   ← boundary O
```

That bottom `O` is safe.

### Step 2: DFS from that boundary `O`

It has no connected interior `O`s, so only that one remains safe.

### Step 3: Flip all other `O`s

```text
X X X X
X X X X
X X X X
X O X X
```



## 🧠 Key Observation

An interior `O` survives **only if** it is connected to the boundary.

That’s the entire problem in one sentence.



## ⚙️ Algorithm

### DFS-based approach

1. Create a visited matrix `vis`
2. Traverse the **first row** and **last row**

   * If a cell contains `'O'` and is not visited, run DFS
3. Traverse the **first column** and **last column**

   * If a cell contains `'O'` and is not visited, run DFS
4. DFS marks all boundary-connected `O`s as visited
5. Traverse the whole board

   * If a cell is `'O'` and not visited, flip it to `'X'`



## ✅ Why this works

Because there are only two types of `O`s:

### 1. Safe `O`s

These are connected to boundary `O`s
We mark them visited and leave them alone

### 2. Captured `O`s

These are not connected to the boundary
So they are truly surrounded and should be flipped

That’s why the solution is correct.



## 🧾 Our Code [C++ / DFS]



```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    void dfs(int row, int col, vector<vector<char>>& board,
             vector<vector<int>>& vis, int dr[], int dc[]) {
        
        vis[row][col] = 1;
        int n = board.size();
        int m = board[0].size();

        for (int i = 0; i < 4; ++i) {
            int nrow = row + dr[i];
            int ncol = col + dc[i];

            if (nrow >= 0 && nrow < n &&
                ncol >= 0 && ncol < m &&
                !vis[nrow][ncol] &&
                board[nrow][ncol] == 'O') {
                dfs(nrow, ncol, board, vis, dr, dc);
            }
        }
    }

public:
    void solve(vector<vector<char>>& board) {
        int n = board.size();
        if (n == 0) return;

        int m = board[0].size();
        if (m == 0) return;

        int dr[] = {-1, 0, 1, 0};
        int dc[] = {0, 1, 0, -1};

        vector<vector<int>> vis(n, vector<int>(m, 0));

        // Traverse top and bottom rows
        for (int j = 0; j < m; ++j) {
            if (!vis[0][j] && board[0][j] == 'O')
                dfs(0, j, board, vis, dr, dc);

            if (!vis[n - 1][j] && board[n - 1][j] == 'O')
                dfs(n - 1, j, board, vis, dr, dc);
        }

        // Traverse left and right columns
        for (int i = 0; i < n; ++i) {
            if (!vis[i][0] && board[i][0] == 'O')
                dfs(i, 0, board, vis, dr, dc);

            if (!vis[i][m - 1] && board[i][m - 1] == 'O')
                dfs(i, m - 1, board, vis, dr, dc);
        }

        // Flip only those O's which were NOT visited
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < m; ++j) {
                if (!vis[i][j] && board[i][j] == 'O')
                    board[i][j] = 'X';
            }
        }
    }
};
```



## 🐢 Brute Force Approach

A brute-force way would be:

* For each `'O'`, run DFS/BFS
* Check whether that connected component touches the border
* If it doesn’t, flip all cells in that component

### Why this is bad

* repeated traversals
* extra storage for each component
* more complicated logic
* easy to make mistakes

### Time complexity

In worst case, this can get very expensive, around `O((n*m)^2)` depending on implementation.



## ⚡ Optimal Approach

The optimal solution is exactly what we used:

### Boundary DFS / BFS

Instead of checking which regions are trapped, mark which regions are **safe**.

### Steps:

* Start DFS from boundary `O`s
* Mark all reachable `O`s
* Flip all unmarked `O`s

### Time Complexity

`O(n * m)`

### Space Complexity

`O(n * m)` for visited + recursion stack in worst case



## 📊 Complexity Analysis

### Time Complexity

### `O(n * m)`

Why?

* Every cell is visited at most once in DFS
* Final traversal also checks every cell once

So total is linear in the number of cells.



### Space Complexity

### `O(n * m)`

Because of:

* visited matrix
* recursion stack in the worst case

If the board is full of `O`s, recursion stack can go as deep as `O(n*m)` in the worst case.

## 🪄 5-Step Intuition Trick So You Never Forget It

### 1. Ask: who can never be captured?

Boundary `O`s

### 2. Ask: who else survives?

Any `O` connected to boundary `O`

### 3. Mark all survivors

Use DFS/BFS from the border

### 4. Anything unmarked is trapped

Those are surrounded

### 5. Flip the trapped ones

Done ✅

This 5-step frame works almost like muscle memory after a while.



## 🧩 Why boundary traversal is enough

Because surrounded regions, by definition, cannot reach the border.

So if an `O` does reach the border through connected `O`s, it’s not surrounded.
That means all surviving cells can be found **just by starting from the border**.

No need to inspect every component separately.

That’s the elegant part.



## 💡 Tips and Tricks

### 1. DFS or BFS both work

Our code uses DFS. BFS would work just as well.

### 2. Boundary problems often work in reverse

This is a classic “reverse thinking” problem.

Instead of:

* find trapped regions

Do:

* find escape-connected regions

That mindset comes up a lot in graphs.

### 3. Be careful with recursion depth

For very large boards, recursive DFS can risk stack overflow.

In interviews it’s usually okay unless constraints are huge.
In production or strict environments, BFS or iterative DFS can be safer.

### 4. Corners get checked twice

That’s fine because `vis` prevents duplicate work.

### 5. This is a graph problem in disguise

Every cell is a node, neighbors are edges.

Grid + connectivity + DFS/BFS = graph problem.

## ❓ FAQs

### Q1. Why not start DFS from every `O`?

Because that repeats work and complicates checking whether a region touches the border.

Starting from border `O`s is cleaner and more efficient.



### Q2. Why does DFS from border `O`s mark all safe cells?

Because any safe cell must have a path of adjacent `O`s leading to the boundary.

DFS explores exactly those paths.



### Q3. Can this be done without a visited matrix?

Yes.

A common trick is:

* mark safe `O`s temporarily as `'#'`
* then flip remaining `O` to `X`
* then turn `'#'` back to `O`

That avoids extra `vis` space, though recursion stack still exists.



### Q4. Is BFS better than DFS here?

Not really better, just different.

* DFS is shorter to write
* BFS avoids deep recursion risk

Both are valid.



### Q5. What if board is empty?

Our code handles that correctly:

```cpp
if (n == 0) return;
if (m == 0) return;
```

Nice touch 👌



## ✨ Cleaner Alternative Without `vis`

This version marks safe cells directly on the board.

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    void dfs(int row, int col, vector<vector<char>>& board, int dr[], int dc[]) {
        int n = board.size(), m = board[0].size();
        board[row][col] = '#';

        for (int i = 0; i < 4; ++i) {
            int nrow = row + dr[i];
            int ncol = col + dc[i];

            if (nrow >= 0 && nrow < n &&
                ncol >= 0 && ncol < m &&
                board[nrow][ncol] == 'O') {
                dfs(nrow, ncol, board, dr, dc);
            }
        }
    }

public:
    void solve(vector<vector<char>>& board) {
        int n = board.size();
        if (n == 0) return;
        int m = board[0].size();
        if (m == 0) return;

        int dr[] = {-1, 0, 1, 0};
        int dc[] = {0, 1, 0, -1};

        for (int j = 0; j < m; ++j) {
            if (board[0][j] == 'O') dfs(0, j, board, dr, dc);
            if (board[n - 1][j] == 'O') dfs(n - 1, j, board, dr, dc);
        }

        for (int i = 0; i < n; ++i) {
            if (board[i][0] == 'O') dfs(i, 0, board, dr, dc);
            if (board[i][m - 1] == 'O') dfs(i, m - 1, board, dr, dc);
        }

        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < m; ++j) {
                if (board[i][j] == 'O') board[i][j] = 'X';
                else if (board[i][j] == '#') board[i][j] = 'O';
            }
        }
    }
};
```

## ⭐ Alternate BFS Version

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    void solve(vector<vector<char>>& board) {
        int n = board.size();
        if (n == 0) return;

        int m = board[0].size();
        if (m == 0) return;

        vector<vector<int>> vis(n, vector<int>(m, 0));
        queue<pair<int, int>> q;

        int dr[] = {-1, 0, 1, 0};
        int dc[] = {0, 1, 0, -1};

        // Push all boundary 'O' cells into queue
        for (int j = 0; j < m; j++) {
            if (board[0][j] == 'O' && !vis[0][j]) {
                q.push({0, j});
                vis[0][j] = 1;
            }

            if (board[n - 1][j] == 'O' && !vis[n - 1][j]) {
                q.push({n - 1, j});
                vis[n - 1][j] = 1;
            }
        }

        for (int i = 0; i < n; i++) {
            if (board[i][0] == 'O' && !vis[i][0]) {
                q.push({i, 0});
                vis[i][0] = 1;
            }

            if (board[i][m - 1] == 'O' && !vis[i][m - 1]) {
                q.push({i, m - 1});
                vis[i][m - 1] = 1;
            }
        }

        // BFS traversal
        while (!q.empty()) {
            auto [row, col] = q.front();
            q.pop();

            for (int i = 0; i < 4; i++) {
                int nrow = row + dr[i];
                int ncol = col + dc[i];

                if (nrow >= 0 && nrow < n &&
                    ncol >= 0 && ncol < m &&
                    !vis[nrow][ncol] &&
                    board[nrow][ncol] == 'O') {
                    
                    vis[nrow][ncol] = 1;
                    q.push({nrow, ncol});
                }
            }
        }

        // Flip unvisited 'O' to 'X'
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                if (board[i][j] == 'O' && !vis[i][j]) {
                    board[i][j] = 'X';
                }
            }
        }
    }
};
```

Core idea:

* Boundary `'O'` and all connected `'O'` are safe
* Mark them visited using BFS
* Whatever `'O'` remains unvisited is surrounded, so flip it to `'X'`

Time complexity: `O(n * m)`

Space complexity: `O(n * m)`
